# 软件开发任务书 (SOW)

**项目名称：** 工业低代码通用底座 - 可视化页面编辑器模块  
**系统架构师/技术负责人：** 孙宇飞  
**文档版本：** V1.0  
**当前阶段：** 架构设计与技术选型

---

## 一、建设背景与目标

为提升前端页面的交付效率及高度定制化能力，本项目拟开发一套基于 React 生态的拖拽式低代码页面编辑器。该模块需作为通用底座的核心能力之一，允许开发或实施人员通过拖拽标准组件（如表单、表格、图表）、配置属性及编写自定义 JavaScript 脚本，快速构建并发布页面。

---

## 二、核心技术栈规范

- **前端框架：** React + React Router v6
- **UI 物料库：** Ant Design（全面替换 Element UI，适配 React 生态，满足复杂企业级与工业场景）
- **数据可视化：** ECharts（通过 echarts-for-react 封装为物料组件）
- **拖拽引擎：** dnd-kit（负责网格与自由布局的拖拽交互）
- **代码编辑器：** Monaco Editor（内嵌于浏览器，用于编写自定义 JS 脚本及 JSON 查看）
- **状态管理：** Zustand + Immer.js（处理深层嵌套的 JSON Schema 状态树）
- **后端接口与存储：** 配合后端的 Java 25 + Spring Boot 4.0 服务架构，最终页面配置数据必须以 JSON Schema 格式存储于 PostgreSQL 的 JSONB 字段中。

---

## 三、功能模块拆解与开发任务

本项目整体划分为三个独立且协同的核心页面模块：

### 模块一：页面资产管理页 (Management Page)

**任务描述：** 编辑器模块的控制台入口，负责页面实体的生命周期管理。

**功能点：**
- 基于 AntD Table 的页面列表展示（分页、按名称/状态检索）
- 核心操作：新建空白页面、从模板复制、删除、启用/禁用状态切换
- 提供通用的 Axios 拦截器，与后端 RESTful API 进行标准的 CRUD 交互

### 模块二：可视化编辑核心页 (Editor Page)

**任务描述：** 该项目开发比重最大的重度单页应用（SPA），实现「拖拽配置 + 脚本编写」的开发环境。

**功能点：**
- **左侧物料面板：** 按分类提供标准 AntD 组件库与 ECharts 容器的拖拽源
- **中间画布区：** 基于 dnd-kit 接收拖拽事件，实时解析 JSON Schema 并渲染占位组件
- **右侧配置面板：**
  - **属性面板：** 动态渲染当前选中组件的 Props 表单（大小、颜色、数据源绑定等）
  - **脚本面板：** 集成 Monaco Editor，提供 JS 执行沙箱，允许开发人员针对当前组件或页面的生命周期（如 onLoad, onClick）编写自定义脚本
- **顶部工具栏：** 保存（全量 JSONB 提交至 PG 库）、撤销/重做、呼出预览

### 模块三：独立预览与运行页 (Preview Page)

**任务描述：** 纯粹的渲染引擎，负责将 JSON Schema 解析为真正的可交互网页。

**功能点：**
- 路由传参或动态获取指定 ID 的 JSONB 数据
- 实现一套高效的「递归渲染器」，无缝挂载对应组件
- **运行时沙箱：** 解析 JSON 中的自定义脚本字符串，使用 `new Function` 或安全沙箱机制动态注入上下文（Ctx）并执行，确保脚本能正确触发 API 请求或操作组件状态

---

## 四、关键技术攻坚点 (Risk & Challenge)

1. **JSON Schema 标准定义：** 需在开发初期明确一棵页面描述树的标准数据结构（包含组件类型、唯一 ID、嵌套子节点、属性集合、事件脚本映射）。

2. **状态隔离与性能优化：** 编辑器操作频繁，需确保某一个组件的属性修改不会导致整个巨型 JSON 树的全局重绘。

3. **脚本执行的上下文安全：** 自定义 JS 脚本在执行时，必须严格控制注入的全局变量，防止对主程序造成污染或破坏。
